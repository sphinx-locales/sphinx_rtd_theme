配置

主题选项

以下选项可以在你项目的 conf.py 文件中定义，使用 html_theme_options 配置选项。

例如：

html_theme_options = {
    'analytics_id': 'G-XXXXXXXXXX',  #  Provided by Google in your dashboard
    'analytics_anonymize_ip': False,
    'logo_only': False,
    'display_version': True,
    'prev_next_buttons_location': 'bottom',
    'style_external_links': False,
    'vcs_pageview_mode': '',
    'style_nav_header_background': 'white',
    # Toc options
    'collapse_navigation': True,
    'sticky_navigation': True,
    'navigation_depth': 4,
    'includehidden': True,
    'titles_only': False
}

目录的选项

以下选项改变了 toctree 指令生成文档导航的方式。

collapse_navigation

启用该功能后，导航条目无法扩展 – 每个条目旁边的 [+] 图标被移除。

类型

boolean

默认

True

注解

将 collapse_navigation 设置为 False，并在有许多文件和深层文件结构的项目中使用 navigation_depth 的高数值，会导致编译时间过长，并可能导致 HTML 文件的文件大小明显增大。

sticky_navigation

当你滚动页面时，导航与主页面内容一起滚动。

类型

boolean

默认

True

navigation_depth

目录树的最大深度。将此设置为 -1，以允许无限的深度。

类型

integer

默认

4

includehidden

指定导航是否包括隐藏的内容目录–即任何被标记为 :hidden: 选项的 toctree 指令。

类型

boolean

默认

True

titles_only

启用后，页面小标题不包括在导航中。

类型

boolean

默认

False

其他选项

analytics_id

如果指定，Google Analytics 的 gtag.js 就会包含在你的页面中。将该值设置为谷歌提供给你的 ID（如 UA-XXXXXXX 或 G-XXXXXXXXXX）。

类型

string

analytics_anonymize_ip

在谷歌分析中对访问者的 IP 地址进行匿名处理。

类型

boolean

默认

False

canonical_url

这将指定一个 规范的 URL 元链接元素，告诉搜索引擎哪个 URL 应该被列为你的文档的主要 URL。如果你有多个 URL，而你的文档是通过这些 URL 提供的，这就很重要。该 URL 指向文档的根路径，需要一个尾部斜线。

类型

URL

0.6.0 版后已移除: 使用 html_baseurl 代替。

display_version

如果 True，版本号将显示在侧边栏的顶部。

类型

boolean

默认

True

logo_only

只显示 logo 图片，不在侧边栏的顶部显示项目名称

类型

boolean

默认

False

prev_next_buttons_location

显示 Next 和 Previous 按钮的位置。这可以是 bottom，top，both，或 None。

类型

string

默认

bottom

style_external_links

在外部链接旁边添加一个图标。

类型

boolean

默认

False

vcs_pageview_mode

改变了使用 display_github, display_gitlab 等文件时的查看方式。当使用 GitHub 或 GitLab 时，这可以是：blob （默认），edit 或 raw。在 Bitbucket 上，这可以是 view （默认）或 edit。

类型

string

默认

blob 或者 view

style_nav_header_background

改变导航栏中搜索区域的背景。该值可以是任何在 CSS background 属性中有效的东西。

类型

string

默认

#2980B9

文件范围的元数据

以下选项可作为 文件范围的元数据 使用：

github_url

强制 Edit on GitHub 按钮使用配置的 URL。

bitbucket_url

强制 Edit on Bitbucket 按钮使用配置的 URL。

gitlab_url

强制 Edit on GitLab 按钮使用配置的 URL。

其他配置

添加 logo

使用 Sphinx 标准选项 html_logo，你可以设置一个图片文件作为侧边栏顶部的 logo 使用。主题选项 logo_only 也允许在边栏顶部只显示标志。

添加自定义 CSS 或 JavaScript

添加自定义 CSS 或 JavaScript 可以帮助你改变这个主题的外观和感受，而不需要 fork 主题供本地使用。

为了在不破坏现有主题文件的情况下添加自定义 CSS 或 JavaScript，你可以 添加文件以包含在你的文档输出中。

目录的显示方式

目前，左边的菜单将根据你的源文件中定义的任何 toctree 指令来构建。它默认输出 4 级深度，以允许快速浏览主题。如果没有定义 TOC 树，Sphinx 的默认行为是使用页面标题来代替。

需要注意的是，如果你在整个文档中不遵循相同的 reST 标题风格，那么 TOC 树的构建就会不正确，所产生的菜单在渲染时可能不会显示正确的深度。

还要注意，默认情况下，目录是用 includehidden=True 设置的。这允许你在索引文件中用 :hidden: 属性设置一个隐藏的 TOC，这将允许你建立一个 TOC 而不在你的索引中呈现。

在默认情况下，当你滚动时，导航将粘到屏幕上。然而，如果你的 TOC 不够高，它将恢复为静态定位。要完全禁用粘性导航，请改变 sticky_navigation 主题选项。
