开发

主题开发者遵循以下准则进行开发和发布计划。文档作者可以根据所需的浏览器/操作系统组合或依赖性支持来决定哪一个主题版本最适合他们的项目。

支持的浏览器

官方浏览器支持由我们的页面浏览分析中最流行的浏览器/操作系统组合决定。官方支持的组合通常在开发过程中进行测试，并在新版本的主题最终发布前进行测试。

支持的组合的较旧版本，以及一些不太常见的组合，被认为是部分支持。这意味着我们并没有对这些组合进行持续的测试，但是我们希望用户体验能够与支持的组合相媲美。

1.0 新版功能: 增加了支持的浏览器和操作系统组合

有几个浏览器/操作系统组合根本不被主题开发者所支持。不支持的组合不接受测试或开发，我们可能不会接受这些组合的主要贡献。

不支持的浏览器/操作系统组合包括。

Internet Explorer（任何操作系统，版本 <=10）。

不支持。IE11 是最后一个部分支持的版本。我们不对之前的版本进行测试。

Internet Explorer（任何操作系统，版本11）。

我们目前只部分支持 IE11，而且只测试主要的错误。支持将在 2.0.0 版本中删除。

Opera（任何操作系统，任何版本）

社区支持。我们没有收到足够的该浏览器的流量，无法在测试和开发中正式支持它。

路线图

我们目前在发展路线图上有几个版本的计划。每一个版本都注明了向后不兼容的变化、弃用和主要功能。

版本遵循 语义化版本，因此一般建议作者将对 sphinx_rtd_theme 的依赖性固定在低于下一个主版本的版本：

$ pip install "sphinx_rtd_theme<=2.0.0"

1.0.0

计划的发布日期

2021 年 8 月

这个版本将是一个稍微向后不兼容的版本，紧随 0.5.2 版本。它将放弃对 Sphinx 1.6 的支持，这在目前是一个相当老的版本。

这个版本将增加对 Sphinx 4.x 版本系列的正式支持，它解决了 Docutils 最新版本 0.17 版本的错误。

从这个版本开始，在构建时将会发出几个废弃警告：

直接安装已被废弃

支持通过 GitHub 直接安装，不再是建议的安装方法。为了便于维护，编译的 assets 最终将从主题库中删除。这些文件将只包括在 PyPI 上提供的内置包/版本中。

我们计划开始在 PyPI 上更频繁地发布开发版本，这样就不再需要从主题源码库中安装。

已建 assets 暂计划在版本 3.0.0 中删除：

对 HTML4 的支持已被弃用

对 Sphinx HTML4 写入器的支持将在 2.0.0 版本中删除。

1.1.0

计划的发布日期

2021 Q3

我们的目标是在 1.x 版本系列中至少有一个错误修复版本来跟进 1.0.0。1.1 版本将不会增加任何主要功能，而是标志着最后一个针对有旧的依赖性的项目的版本，如 Sphinx 1.8、HTML4 或对 IE11 的必要支持。

2.0.0

计划的发布日期

2022 Q1

这个版本将标志着新一轮功能开发的开始，以及一些向后不兼容的变化和弃用。

值得注意的是，这个版本计划进行以下向后不兼容的修改：

Sphinx 1.x, Sphinx 2.x, 和 Docutils 0.16 将不被测试。

官方将放弃对这些版本的支持，尽管它们可能仍在继续工作。主题开发者将不再测试这些版本。

将取消对 HTML4 的支持

从这个版本开始，我们将只支持 HTML5 写入器的输出，试图使用 HTML4 写入器的构建将失败。如果你还在使用 HTML4 编写器，或者在你的 Sphinx 配置文件中有 html4_writer = True 选项，你需要删除这个选项，或者将你的依赖关系固定在 sphinx_rtd_theme<=2.0.0，直到你可以为止。

这个选项在过去被建议用来解决 HTML5 支持的问题，不应该再这个主题和 Sphinx 的现代组合所需要。

3.0.0

这个版本还没有计划，然而有计划在 2.0 之后的版本中可能用 Bootstrap 取代 Wyrm。

这个版本还暂时计划从我们的资源库中删除内置的 CSS 和 JavaScript assets。这将取消直接从 GitHub 安装软件包的能力，而建议用户从 PyPI 安装开发版本
